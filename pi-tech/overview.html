Note: F4 and F5 are closely related classes. F4 covers the broad technology sector; F5 covers telecoms specifically. Many programmes cover both under a single F4/F5 policy. Cyber liability (CY/CZ) and tech E&O are distinct but frequently overlap — tech E&O covers failure to provide services correctly; cyber covers malicious external attack.

Coverage Type Cards

1. Software Errors & Bugs — F4

Description: Financial loss to third parties caused by defects, errors or failures in software delivered by the insured.

Scope: Code bugs causing wrong calculations, data corruption, system crashes, API errors.

Exclusions: Physical damage and bodily injury (covered by general liability).

Key perils: Financial software errors (Knight Capital), trading system outages, ERP misconfiguration.

Coverage trigger: Third party suffers quantifiable economic loss as direct result of insured's software error.

2. IT Project & Implementation Failure — F4

Description: Covers claims arising from failed, delayed or over-budget IT projects where the technology provider bears professional responsibility.

Common scenarios: ERP implementation failure (SAP/Oracle), system migration errors (TSB), IT outsourcing project abandonment.

Coverage: Wasted client expenditure, business interruption losses, costs of rectification by replacement vendor.

Key challenge: Claims often involve significant causation disputes — whether failure is client or vendor's fault.

3. SaaS / Cloud Platform E&O — F4

Description: Covers SaaS and cloud service providers for errors causing customer losses.

Key perils: Service outages beyond SLA — if downtime causes client financial loss exceeding contractual remedy. Data loss or corruption on provider's platform. Feature misconfiguration or API error causing client workflow failure.

SLA distinction: Planned outage with notice = likely excluded; unplanned outage without warning = covered.

Paradigm case: CrowdStrike July 2024 Falcon sensor update — defective content configuration update to 8.5 million Windows devices caused global systemic outage.

4. Managed Services & IT Consulting — F4

Description: Covers IT managed service providers (MSPs), IT consultants and systems integrators for professional negligence.

MSP perils: Failure to maintain client systems, missed patch management causing breach, backup failure preventing recovery, helpdesk misadvice.

Consulting perils: Negligent architecture advice, technology selection negligence, security audit failures.

Key exposure: MSP acting as gatekeeper to client systems bears significant fiduciary-style responsibility.

5. Telecom Network E&O — F5

Description: Professional indemnity for telecoms operators arising from network design, provisioning and maintenance errors.

Key perils: Billing errors (overcharging systemic — large aggregate exposure), number porting errors, ISDN/VoIP configuration errors causing 999 access failure (highest severity), leased line provisioning errors.

Regulatory exposure: Ofcom General Conditions require telecoms providers to maintain emergency call access — breach creates both regulatory and PI exposure.

6. Telecom Managed Services & MVNO — F5

Description: MVNOs (Mobile Virtual Network Operators), resellers and telecom managed services providers.

Coverage: Billing platform errors, wholesale billing disputes, roaming misconfiguration, service quality failures under SLA.

MVNO exposure: MVNO uses host network under wholesale agreement — PI may overlap with host network operator's own policy. Careful programme coordination required to avoid gaps.

Tech E&O vs Cyber Insurance

Understanding the Coverage Interface

These products frequently overlap but cover different root causes:

Tech E&O (F4/F5): Covers the insured's own errors/omissions in providing technology services. Triggered by: software bugs, project failures, service outages, negligent advice. The insured is at fault.

Cyber (CY/CZ): Covers losses arising from external cyber attacks, data breaches, ransomware. The insured is a victim. Triggered by: hacker intrusion, phishing, supply chain compromise.

Overlap zone: CrowdStrike-type event — update pushes defective code causing outage. Tech E&O? Or cyber? Debate ongoing in market 2024. Generally: if failure is own code error = Tech E&O; if failure is external attack exploiting own code = Cyber.

Coordinated programmes: Most sophisticated tech buyers purchase both F4/F5 and CY/CZ with back-to-back coordination clauses.
