⚙️ Section 3 — JavaScript & Interactivity Pending

No JS errors in browser console (F12) on any page
Open DevTools → Console on each page. Any red errors must be investigated and fixed before sign-off.
Critical
Chart.js chart renders with visible data (rates-analysis.html)
The rate index line chart must show data points. If blank, check: (1) CDN loaded, (2) canvas ID matches JS, (3) no ctx.label in element callbacks — use ctx.chart.data.labels[ctx.dataIndex] instead.
Critical
Chart tooltips work on hover
Hover over chart data points. Tooltip should show year, index value, and loss event annotation where applicable.
Major
Database filters work correctly (database.html)
Test each filter dropdown (type, basin, era) and the search box. Row count should update. Clear button resets all.
Critical
Database column sort works on all sortable columns
Click year, loss, and fatality column headers. Data should re-sort. Check sort direction toggles correctly.
Major
No unescaped apostrophes in JS string literals
Any word like "Lloyd's", "O'Brien", or curly apostrophes (') inside single-quoted JS strings will cause a syntax error. Use Unicode escapes (\u2019) or double quotes instead.
Critical
localStorage used for build-tracker checkboxes only
No other page should persist state to localStorage. If used elsewhere, confirm it degrades gracefully if localStorage is unavailable.

📝 Section 4 — Content Quality Pending

Module 01 (History): Eras cover key periods with named events
Should span at least 60 years. Key market-shaping events named with approximate dates. Major underwriters or syndicates referenced.
Major
Module 02 (Timeline): At least 10 entries spanning 20 years
Mix of loss events and market events. Colour-coded by event type. Stats panel includes loss figure and fatalities where relevant.
Major
Module 03 (Database): At least 25 records; all loss figures in USD
All dollar amounts should be consistent (approximate insured, not economic). Source of data noted in page text. Filter and sort verified (Section 3).
Major
Module 04 (Underwriting): Worked example present with realistic figures
Rates, TIVs, and premiums must be in the right order of magnitude for the class. All coverage sections (S1–S4 for energy; equivalent for other lines) addressed.
Critical
Module 05 (Rates): Rate chart covers at least 20 years; ROE cards complete
All three ROE scenarios (bull/base/bear) must have loss ratio, expense ratio, combined ratio, underwriting profit, and final ROE figure. Combined ratios must add up correctly.
Critical
Module 05 (Rates): ROE arithmetic verified
Check: (1) Combined = loss ratio + expense ratio. (2) UW profit = GWP × (1 – combined/100). (3) ROE = (UW profit + investment income) / capital base × 100. All three scenarios.
Critical
Module 06 (Risk Mitigation): Key regulatory standards named correctly
Regulatory bodies, standards, and certification names must be accurate (e.g. NORSOK D-010, not NORSEK; IWCF not IWFC; NSTA not NsTa).
Major
Module 07 (Global Programmes): Key basins covered with fronting status correct
Jurisdictions where Lloyd's cannot write direct (Brazil, Nigeria, Angola, Indonesia) must be flagged as requiring fronting. Do not mark direct-access markets as requiring fronts.
Major
All Lloyd's risk codes for this hub listed correctly throughout
Check hub index page, footer, hub-index.json, and header tag. Codes must match the active Lloyd's risk code list exactly.
Critical
No placeholder or "lorem ipsum" content anywhere
Search the folder for "TBD", "TODO", "lorem", "placeholder", "FIXME". None should appear in final files.
Critical

🔢 Section 5 — Data Accuracy Spot-Checks Pending

Major loss figures are in the right order of magnitude
Cross-check 2–3 headline loss figures against a known reference (e.g. Macondo ~$4bn insured; Piper Alpha ~$1.4bn; 2005 hurricane ~$9bn). Figures should not be off by 10× or more.
Critical
Dates of key events are correct (year at minimum)
Piper Alpha: 1988. Macondo: 2010. Ivan: 2004. Katrina/Rita: 2005. If any year is wrong, update across all modules consistently.
Major
Regulatory body names and jurisdictions are correct
E.g. BSEE (not BOEM) regulates drilling safety. NSTA (not OGA) is the current UKCS regulator (renamed 2022). PSA Norway (not NPD) regulates safety on NCS.
Major
Underwriting rates and loss ratios are in plausible ranges
Offshore property rates: typically 0.1%–0.5% of TIV. OEE: 1%–4% of TIV. Liability: per-occurrence premium. Loss ratios: 50%–120% are plausible; >200% would require explanation.
Major
Worked example premium components add up correctly
Manually verify the total premium in the worked example equals the sum of all component premiums. Small rounding OK (<1%); larger discrepancies must be fixed.
Critical
